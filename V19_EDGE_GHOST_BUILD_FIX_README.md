# V19 贴边稳定性 / 透明框 / 构建脚本修复

## 修复内容

1. 一键构建脚本
   - `build.bat` 现在会自动切换到项目目录，双击运行不再受当前工作目录影响。
   - 自动检查 Node.js / npm / Rust / Cargo。
   - 构建日志输出到 `build.log`。
   - 新增 `build-dev.bat` 用于开发运行。
   - 保留中文入口 `一键构建.bat`。

2. 贴边隐藏第一次触发弹出又缩回
   - Rust 原生贴边状态机新增 reveal hold 保护。
   - 从触发条展开或自动回弹后，会等待鼠标真正离开主窗口后才允许再次隐藏。
   - 避免 `hideDelay=0` 时展开后立刻被 Visible 分支再次隐藏。

3. 透明框残影开关
   - 设置 → 操作 → 动画方式 下方新增：`隐藏后强制清理透明框残影`。
   - 开启后，隐藏动画结束会 `SW_HIDE` 并把隐藏的主窗口停放到屏幕外，减少透明 WebView + AnimateWindow 偶发残影。
   - 如果该方法影响动画手感，可以关闭该开关对比。

## 推荐设置

如果你的视频里那种透明框仍出现，建议优先测试：

- 动画方式：`SetWindowPos 无残影滑动（推荐，EaseOutQuint）`
- 贴边动画速度：`70~100ms`
- 隐藏后强制清理透明框残影：开启

如果想测试系统动画：

- 动画方式：`AnimateWindow 系统滑动`
- 隐藏后强制清理透明框残影：先开启，如果还有问题再关闭对比。
