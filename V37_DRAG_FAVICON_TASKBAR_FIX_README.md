# V37 拖动粘连、网站图标与任务栏保留修复

## 本版修改

1. 子目录/标签页里的项目拖动再次加固：
   - 只有鼠标左键仍然按住，并且达到“设置 - 拖动”里的长按时间后，才会把项目交给拖动排序。
   - pointerup / pointercancel / mouseup / 失焦都会清理拖动等待状态，避免单击打开后项目粘在鼠标上。
   - 给项目按下时加入 pointer capture，降低鼠标移出项目后丢失释放事件的概率。

2. 添加网站弹窗：
   - 名称和网址仍在同一个软件内主题弹窗里填写。
   - 新增“这次添加时自动获取网站图标”选项，只对当前添加的网址有效。
   - 未写 http/https 时仍会自动补 https://。

3. 网站 favicon：
   - 新增后端命令 fetch_website_favicon。
   - 添加网站或刷新本页图标时，网址项目可从网站获取 favicon。
   - 在“设置 - 图标”新增“自动获取网站 favicon 时保存到本地”开关。
   - 开启后图标保存到本地缓存，并把本地图片路径写入项目图标；关闭后使用在线 favicon 地址。

4. 侧边隐藏时任务栏图标保留：
   - Windows 下给主窗口追加 WS_EX_APPWINDOW，并移除 WS_EX_TOOLWINDOW。
   - 主窗口显示/隐藏、原生贴边循环重新找到窗口时都会重新应用该样式。

## 构建

已执行：

```bash
npm run build
```

前端构建通过。
