# V33 单击启动项目误拖拽修复

本版本针对用户反馈重新定位问题：

> 开启“单击启动”后，在子目录里的项目上单击打开项目，项目会吸附到鼠标上。

## 修复内容

1. 修复的是“子目录里的项目”，不是左侧标签/特殊标签。
2. 单击启动模式下，项目排序拖拽改为长按触发：
   - 普通单击会在拖拽 delay 结束前完成，不会进入 dnd-kit 拖拽态。
   - 需要排序时按住项目约 220ms 再拖动。
3. 单击启动模式下点击容差从 6px 提高到 10px，降低手抖导致的误判。
4. 右键不再传入排序拖拽监听，避免右键菜单与项目排序传感器互相干扰。

npm run build
