# V29 - 便签右键菜单定位与光标保持修复

本版本在 v28 基础上修复两个便签右键菜单问题：

1. 右键菜单边界定位
   - 优化通用 `useSmartMenuPosition`。
   - 当菜单会超出当前 exe/webview 可见区域时，自动优先显示到鼠标的另一侧：右侧不够则向左，底部不够则向上。
   - 同时继续保留边界夹紧，避免菜单贴边溢出窗口。
   - 对主界面缩放后的菜单位置也做了更稳的视口计算。

2. 便签右键不再改变输入光标位置
   - 右键按下时先保存当前 textarea 光标/选区/滚动位置。
   - 阻止右键 mousedown 的默认选区移动行为。
   - 打开右键菜单后恢复原来的输入光标位置，菜单插入分割线仍然按鼠标所在行插入。
   - 菜单本身的 mousedown 也阻止默认聚焦变化，避免点菜单时抢走便签光标。

已执行：

```bash
npm run build
```

前端构建通过。
